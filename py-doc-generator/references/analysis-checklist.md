# Python 代码分析检查清单

在阶段 1 中使用此检查清单，系统地从每个 Python 文件中提取信息。

## 逐文件提取

对每个 `.py` 文件，提取以下内容：

### 模块级
- [ ] 模块 docstring（如果存在）
- [ ] 所有 import（分类：标准库、第三方、内部）
- [ ] 模块级常量（`UPPER_CASE` 变量）
- [ ] 模块级函数（非类内函数）
- [ ] `__all__` 导出列表（如果定义了）

### 类
- [ ] 类名和父类
- [ ] 类装饰器（`@dataclass`、`@frozen`、`@define` 等）
- [ ] 类 docstring
- [ ] 类属性及类型标注
- [ ] 属性默认值
- [ ] `__init__` 参数（非 dataclass 时）
- [ ] 公共方法（不以 `_` 开头）
  - [ ] 完整签名（含类型标注）
  - [ ] Docstring（Args、Returns、Raises）
  - [ ] 装饰器（`@staticmethod`、`@classmethod`、`@property`、`@abstractmethod`）
- [ ] 属性（`@property` 装饰的方法）
- [ ] 关键私有方法（以 `_` 开头但逻辑重要的）

### 独立函数
- [ ] 函数名
- [ ] 完整签名（含类型标注）
- [ ] Docstring
- [ ] 装饰器

### 枚举
- [ ] 枚举名和基类（`str`、`int`、`Enum`）
- [ ] 所有成员及其值
- [ ] Docstring

### 类型别名
- [ ] `TypeAlias` 定义
- [ ] `TypedDict` 定义
- [ ] `Protocol` 定义

## 关系提取

### 继承关系
- [ ] 哪些类继承自什么
- [ ] Mixin 类
- [ ] 抽象基类

### 组合关系
- [ ] 作为其他类属性的类
- [ ] 构造函数注入（参数类型为其他类）

### 依赖关系
- [ ] 哪些模块导入了哪些
- [ ] 外部服务依赖（数据库、缓存、消息队列）
- [ ] 配置依赖

## 业务逻辑识别

### 入口点
- [ ] CLI 入口（`if __name__ == "__main__"`、click 命令）
- [ ] HTTP 路由处理器（Flask `@app.route`、FastAPI `@router`）
- [ ] 事件处理器/信号处理器
- [ ] 后台任务定义

### 关键流程
- [ ] 端到端追踪 3-5 个关键业务流程
- [ ] 识别函数/方法调用序列
- [ ] 标记分支逻辑（错误处理、验证、备选路径）

### 数据流
- [ ] 数据从哪里进入系统
- [ ] 经过了哪些转换
- [ ] 数据存储/检索位置
- [ ] 数据从哪里离开系统（响应、导出、通知）

## 架构模式检测

查找以下常见模式：

| 模式 | 特征 |
|---|---|
| MVC | `models/`、`views/`、`controllers/` 目录 |
| 分层 | `service/`、`repository/`、`api/` 分离 |
| 六边形 | 端口与适配器，`inbound/`、`outbound/` |
| 整洁架构 | 实体、用例、接口适配器 |
| 事件驱动 | 事件总线、消息队列、发布/订阅装饰器 |
| 插件 | `plugins/` 或 `extensions/` 目录，钩子注册 |
| 管线 | 处理器链、中间件栈 |

## 模块分类

为每个模块分配以下角色之一：

| 角色 | 说明 |
|---|---|
| `entry` | 应用入口、CLI、服务启动 |
| `model` | 数据模型、ORM 模型、Schema |
| `service` | 业务逻辑、领域服务 |
| `controller` | 请求处理、路由定义 |
| `repository` | 数据访问、数据库查询 |
| `middleware` | 请求/响应处理管线 |
| `config` | 配置管理 |
| `utility` | 辅助函数、共享工具 |
| `adapter` | 外部服务集成 |
| `test` | 测试文件和 fixtures |
